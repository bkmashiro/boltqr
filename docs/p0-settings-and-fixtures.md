# BoltQR P0 Settings + Password Page Fixture Corpus

## Settings page

Build output:

- `dist-extension/extension/options.html`
- `dist-extension/options.js`
- `dist-extension/assets/options-*.css`

Manifest entries:

```json
{
  "options_page": "extension/options.html",
  "action": {
    "default_popup": "extension/options.html"
  }
}
```

User-facing settings:

- helper endpoint, default `http://127.0.0.1:17321`
- bearer token
- enable/disable candidate handoff
- manual-only handoff toggle
- health check button against `/healthz`

Storage keys:

```txt
smartExtractEndpoint
smartExtractToken
smartExtractEnabled
smartExtractManualOnly
```

## Fixture corpus

Generated by:

```bash
pnpm fixtures
```

Output:

```txt
fixtures/password-pages/
  manifest.json
  01-visible-extract-password.html
  02-pan-code-and-archive-password.html
  ...
  12-non-url-qr-with-password.html
```

The 12 cases cover:

1. visible `解压密码`
2. extraction code + archive password on same page
3. copy button `data-clipboard-text`
4. hidden input value
5. QR-nearby password text
6. English `Archive password`
7. hostname-as-password fallback
8. `title` attribute
9. `aria-label` attribute
10. noisy page with lots of junk tokens
11. Quark-style `访问码`
12. non-URL QR text with page password
